**Prototype Design Document v2.1.0**
2026-04-10

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 장르 | 싱글 플레이 실시간 액션 RPG |
| 플랫폼 | 모바일 (Android / iOS), PC 테스트 지원 |
| 타겟 유저 | 지하철·버스 등 통학·출퇴근 시간에 즐기는 숏폼 액션 유저 |
| 화면 방향 | 세로 모드 (Portrait) |
| 렌더 파이프라인 | URP 2D |
| 조작 | 전체 화면, 단일 손가락(엄지) / PC 마우스 지원 |
| 클리어 조건 | 스테이지 내 모든 보스 처치 |
| 프로토타입 목표 | 핵심 전투 메커니즘 검증 및 재미 확인 |

### 디자인 철학

- **액션 쾌감**: 단일 손가락 조작으로 완성하는 직관적 전투.
- **명확한 턴**: 이동-공격 간 간섭을 제거한 피지컬 전투.
- **공정한 BM**: 무소과금도 실력으로 극복 가능하되, 과금 시 압도적 쾌적함을 제공하는 구조.

## 2. 게임 구조

### 2-1. 스테이지 / 방 구조 (나선비경 스타일)

게임 진행은 **스테이지** 단위로 이뤄진다. 한 스테이지는 4개의 보스 방과, 그 사이마다 배치된 **인터미션 방**으로 구성된다.

```
[보스1] → [인터미션] → [보스2] → [인터미션] → [보스3] → [인터미션] → [보스4] → [클리어]
```

#### 인터미션 방 (Intermission Room)

- HP 100% 회복(자동) + 스탯 선택지 2개 중 1개 선택
- 회복은 기본 적용, 스탯 선택은 능동적 선택
- 자세한 스탯 선택 규칙은 §8 참조

#### 진행 / 사망 / 이어하기

- 보스 방에서 사망 시: **죽은 보스부터 이어하기 가능** (해당 스테이지 진행 상태 유지)
- 사망 후 사용자가 능동적으로 **로비로 나갈 수도 있음** → 로비 복귀 시 **무조건 진행도 초기화** (해당 스테이지의 모든 진척, RunStat 소멸)
- 스테이지 클리어 후: 보상 화면 → **로비 복귀 / 다음 스테이지 진입** 사용자 선택

#### 스테이지 내 진행 표시

- HUD에 현재 위치 표시 (예: `1 / 4`, `2 / 4`)
- 다음 방 미리보기는 **없음** (긴장감 유지)

#### 스테이지 보유 수

프로토타입 단계에서는 **10개 스테이지**를 제작한다. 이후 지속적으로 추가한다.

### 2-2. 보스 패턴

모든 보스의 공격은 **시퀀셜 구조**로 정의된다. 보스는 **단일 페이즈, 2 페이즈, 3 페이즈** 등 다양한 구조를 가질 수 있다 (보스마다 다름).

자세한 보스 라인업은 §5-3 참조.

### 2-3. 공격 예고 인디케이터

Windup 구간 동안 아웃라인(범위 경계 고정) + 스캔라인(중심→경계 확장)이 표시된다.
스캔라인이 경계에 닿는 순간 = 공격 발동.

| 모양 | 용도 |
|------|------|
| 부채꼴 (Sector) | 근접 광역 공격 |
| 사각형 (Rectangle) | 돌진 등 직선 범위 공격 |

- `angleOffset` 필드로 여러 방향 인디케이터 동시 표시 가능.
- `lockIndicatorDirection`: true 시 Windup 시작 방향 고정.
- 인디케이터 범위 = 히트 판정 범위 (동일 데이터 사용).

## 3. 조작 체계

입력 유효 영역: **전체 화면**. 단일 손가락(엄지) 기준. PC에서는 마우스로 에뮬레이션.

### 3-1. 기본 입력

| 입력 | 상태 | 조건 | 설명 |
|------|------|------|------|
| Drag | Move | 이동 거리 ≥ 20px, 지속 ≥ 0.3초 | 가상 조이스틱과 함께 이동. 공격 판정 없음 |
| Tap | Attack | 이동 거리 < 20px, 지속 < 0.2초 | 전방 사각형 범위 광역 공격. 즉시 히트 판정. 인디케이터 0.4초 표시. 딜레이 중 연타 차단 |
| Swipe | Dodge | 이동 거리 ≥ 20px, 지속 < 0.3초 | 입력 벡터 방향으로 회피 대시. 대시 전 구간 무적. 쿨다운 2초 |
| Hold → Release | Charge → Skill | 정지 상태, 지속 ≥ 0.3초 | 홀드 중 게이지 충전, 손을 떼면 차지 스킬 발동 |

### 3-2. 입력 차단 규칙

특정 액션 진행 중에는 다른 입력이 차단된다. **차단된 입력은 큐잉되지 않고 즉시 무시**된다.

| 진행 중 액션 | 차단되는 입력 |
|-------------|--------------|
| 회피 대시 | Tap |
| 저스트 회피 슬로우 | Tap |
| 고유 스킬 발동 ~ 복귀 | Tap |
| 차지 스킬 발동 | Tap |
| 회피 쿨다운 (2초) | Swipe |

### 3-3. 저스트 회피 시스템

저스트 회피: **전 캐릭터 공통**. 회피 대시 중 적의 공격을 피격 시 발동. 한 회피당 1회 한정.

| 단계 | 설명 |
|------|------|
| 발동 조건 | 회피 대시 중(JustDodgeAvailable == true) 적 공격 피격 시 발동 |
| 슬로우 모션 | 발동 시 일정 시간 게임 속도 감소 (AnimationCurve 기반, 기본 3초) |
| 카메라 줌 | 발동 시 카메라가 플레이어를 향해 줌인 후 복귀 |
| 무적 구간 | 슬로우모션 구간 전체 무적 유지 |
| 고유 스킬 발동 | 슬로우 중 Hold 입력 시 캐릭터 고유 스킬 즉시 발동 |

## 4. 캐릭터 설계

### 4-1. 공통 기본 스탯 (베이스라인)

아래 수치는 메타 스탯 / 런 스탯이 적용되기 전의 **기본값**이다.

| 항목 | 수치 | 비고 |
|------|------|------|
| 최대 HP | 500 | 일반 적 10회, 보스 일반 공격 약 6회에 사망 |
| 기본 공격력 | 50 (사각형 범위 광역) | 일반 적 5타, 보스 40타 처치 기준 |
| 이동속도 | 5 | 화면 절반을 약 1초에 이동 |
| 무적 구간 | 회피 대시 전 구간 | unscaledTime 기반 |
| 회피 쿨다운 | 2초 | 쿨다운 중 Swipe 입력 차단. HUD 세로 게이지로 표시 |

### 4-2. 캐릭터별 고유 메커니즘

기본 입력(Tap/Drag/Swipe/Hold)의 동작은 §3 조작 체계를 따른다.
아래는 각 캐릭터가 기본 시스템과 다르게 동작하는 부분만 기술한다.

#### ⚔️ 전사 (Warrior) — '인내와 패링' *(미구현)*

- **홀드 중**: 방패 방어 (데미지 감소, 버팀)
- **홀드 후 릴리즈 (차지 스킬)**: 대지 분쇄 — 전방 광역 공격
- **홀드 후 스와이프**: 해당 방향으로 방패 밀쳐내기 (데미지 + 넉백). 피격 중이었다면 **패링(저스트 회피) 판정** 발동
- **저스트 회피(패링) 후 고유 스킬**: 즉시 대지 분쇄 (차지 시간 없이)

#### 🗡️ 암살자 (Assassin) — '잔상 스태킹과 난무' *(미구현)*

- **회피**: 일반 회피 동작 동일 (특별 효과 없음)
- **저스트 회피 시 고유 스킬**: 회피 전 위치에 잔상 생성 (피해 없음, 어그로 없음). 잔상 활성 중 본체의 모든 공격(Tap, 차지 스킬)에 잔상이 동시에 동참
- **차지 스킬 (홀드 → 릴리즈)**: 360도 원형 광역 베기

#### 🤺 레이피어 (Rapier) — '빌드업과 수확' *(구현 완료)*

- **회피**: 일반 회피 동작 동일 (특별 효과 없음)
- **저스트 회피 후 고유 스킬**: 가장 가까운 적에게 대시 → 사각형 범위 내 전체 적에게 표식 1중첩 부여 + 데미지 → 회피 목적지(DodgeDest)로 복귀. 표식 최대 5중첩
- **차지 스킬 (홀드 → 릴리즈)**: 표식이 있는 모든 적에게 각 보유 표식 중첩 수 × (공격력 × chargeMarkMultiplier) 데미지. 이후 모든 표식 소비
- **무적 구간 추가**: 스킬 대시~복귀 구간 전체 무적. 스킬 및 회피 진행 중 일반 공격 차단

#### 🏹 사냥꾼 (Ranger) — '거리 조절과 화망' *(미구현)*

- **Tap**: 기본 공격이 원거리 사격으로 대체
- **회피**: 입력 벡터 방향으로 대시 + 회피 지점에 지뢰 설치
- **저스트 회피 후 고유 스킬**: 즉시 강화 폭발 화살 발사 (차지 시간 없이)
- **차지 스킬 (홀드 → 릴리즈)**: 일직선 관통 화살. 시전 중 적에게 경직 부여

### 4-3. 구현 상태

현재 (2026-04-10) **레이피어만 구현**되어 있다. 나머지 3 캐릭터는 캐릭터 선택 화면에서 "Coming Soon" 잠금 처리한다.

## 5. 적 설계

### 5-1. 일반 적

| 항목 | 수치 | 비고 |
|------|------|------|
| 최대 HP | 250 | 광역 공격 고려 상향. 단일 기준 플레이어 5타 |
| 공격력 | 50 | 플레이어 10회 피격 사망 기준 |
| 이동속도 | 2.5 | 플레이어의 절반 속도 |
| 공격 주기 | 1.5초 | 타이밍 학습 가능한 주기 |
| 공격 예고 | 0.5초 Windup | 아웃라인 + 스캔라인 인디케이터로 시각 예고 |
| 접근 방식 | 분산 접근 | 각자 다른 각도에서 접근. 뭉치기 방지 |

### 5-2. 보스 (공통 기본값)

| 항목 | 수치 | 비고 |
|------|------|------|
| 기본 최대 HP | 2000 | 베이스라인. 위치/스테이지에 따라 스케일링 |
| 기본 공격력 | 80 | 베이스라인 |
| 광역 스킬 데미지 | 120 | 기본값 |
| 이동속도 | 1.5 | 느리고 무거운 움직임 |

#### 보스 스케일링 공식

보스의 HP / 공격력은 **스테이지 내 위치**와 **스테이지 번호**에 따라 다중 스케일링된다.

##### 스테이지 내 위치 배수 (1 → 4번 보스)

| 위치 | HP 배수 | ATK 배수 | 비고 |
|------|---------|----------|------|
| 1/4 | × 1.0 | × 1.0 | 도입 |
| 2/4 | × 1.3 | × 1.05 | 1회 스탯 누적 후 |
| 3/4 | × 1.7 | × 1.10 | 2회 스탯 누적 후 |
| 4/4 | × 2.2 | × 1.20 | 스테이지 보스 격 |

##### 스테이지 간 배수 (Stage N)

```
HP_multiplier  = 1 + 0.25 × (N - 1)
ATK_multiplier = 1 + 0.10 × (N - 1)
```

##### 최종 보스 능력치

```
HP  = 2000 × Position_HP × Stage_HP
ATK = 80   × Position_ATK × Stage_ATK
```

| 예시 | HP | ATK |
|------|-----|-----|
| Stage 1 / 1번 보스 | 2000 | 80 |
| Stage 1 / 4번 보스 | 4400 | 96 |
| Stage 5 / 4번 보스 | 8800 | 134 |
| Stage 10 / 4번 보스 | 14300 | 182 |

수치는 프로토타입 단계의 시작점이며, 플레이테스트 후 조정한다.

### 5-3. 보스 라인업 (7종)

각 보스는 플레이어가 다른 스킬을 쓰도록 강제하는 것을 목표로 디자인되었다.

| # | 이름 | 컨셉 | 학습 목표 | 페이즈 | 사용 AttackAction |
|---|------|------|----------|--------|------------------|
| 1 | **Titan** | 무거운 거인 | 회피 타이밍, 안전 거리 | 2 | Melee, Charge |
| 2 | **Specter** | 텔레포트 암살자 | 위치 예측 | 2 | Teleport, Melee |
| 3 | **Pyromancer** | 마법사형 원거리 | 거리 좁히기, 장판 회피 | 2 | Projectile, GroundHazard, MultiDirectional |
| 4 | **Berserker** | 빠른 근접 콤보 | 콤보 사이 회피 타이밍 | 2 | Melee×3~4 콤보, Charge |
| 5 | **Stormcaller** | 다방향 동시 공격 | 안전 방향 판단 | 3 | MultiDirectional, Projectile (유도) |
| 6 | **Gravekeeper** | 미니언 소환사 | 우선순위 판단, 미니언 처리 | 2 | Summon, Melee |
| 7 | **Twin Phantoms** | 2체 동시 보스 | 다중 위협 관리 | 1 | Melee × 2, 본체-분신 협공 |

#### 보스 사용 규칙

- 한 스테이지 내에서 같은 보스는 **재출현하지 않음**
- 같은 보스를 다른 스테이지에서 **강화 버전으로 재활용 가능**
- 새 보스는 지속적으로 추가됨

#### 스테이지 1 배치 (학습 곡선)

| 위치 | 보스 | 의도 |
|------|------|------|
| 1/4 | Titan | 가장 단순. 튜토리얼 |
| 2/4 | Berserker | 콤보 압박 학습 |
| 3/4 | Pyromancer | 원거리 보스 학습 |
| 4/4 | Specter | 1스테이지 보스 격 |

#### 신규 AttackAction 종류

기존 4종 (`Melee`, `Aoe`, `Charge`, `Teleport`)에 다음을 추가한다. **단, 해당 기믹을 사용하는 보스를 구현할 때 함께 추가한다.** 미리 만들지 않는다.

| 종류 | 설명 | 사용 보스 |
|------|------|----------|
| Projectile | 투사체 발사 (직선/곡선/유도) | Pyromancer, Stormcaller |
| GroundHazard | 지면 장판 (일정 시간 데미지 존) | Pyromancer |
| MultiDirectional | 다방향 동시 공격 (회피 방향 강제) | Pyromancer, Stormcaller |
| Summon | 미니언 소환 | Gravekeeper |

## 6. 능력치 시스템

### 6-1. 사용 능력치 (프로토타입)

프로토타입 단계에서는 다음 능력치만 사용한다. 추후 확장 가능.

| 능력치 | 약어 | 단위 |
|--------|------|------|
| 최대 HP | HP | 정수 |
| 공격력 | ATK | 정수 |
| 이동속도 | MS | 실수 |
| 회피 쿨다운 감소 | DodgeCDR | % |
| 차지 시간 단축 | ChargeTimeReduction | % |
| 무적 시간 증가 | InvincibilityBonus | % |
| 크리티컬 확률 | CritChance | % |
| 크리티컬 데미지 | CritDamage | % |
| 쿨타임 감소 | CDR | % |
| 스킬 데미지 증가 | SkillDamage | % |

장신구 등에서 위 능력치들이 랜덤하게 부여된다.

### 6-2. 메타 스탯 vs 런 스탯

능력치는 **반드시 두 종류로 명확히 분리**되며, 코드 상으로도 다른 네임스페이스/클래스로 관리한다.

| 종류 | 정의 | 출처 | 저장 | 적용 시점 |
|------|------|------|------|----------|
| **MetaStat** | 영구 능력치 | 캐릭터 레벨, 장비, 룬 | JSON 영구 저장 | 게임 시작 시 캐릭터 모델에 주입 |
| **RunStat** | 일회성 능력치 | 인터미션 방의 스탯 선택 | 메모리 only, 스테이지 종료 시 소멸 | 스테이지 진행 중 누적 적용 |

### 6-3. 계산식

```
최종 능력치 = (기본값 + MetaStat 깡합) × (1 + MetaStat % 합) × (1 + RunStat % 합) + RunStat 깡합
```

- 기본값: §4-1의 베이스라인 (HP 500, ATK 50, MS 5)
- 깡합과 % 합을 분리한 후, RunStat 깡합을 마지막에 더한다
- 동일 능력치의 깡 / % 부스트가 모두 가능

#### 예시

```
기본 HP 500
+ 장비 깡 HP 200
+ 장비 % HP +20%
+ 런 % HP +25% (스탯 1회 선택)
+ 런 % HP +25% (스탯 2회 선택, 누적)
+ 런 깡 HP +0
= (500 + 200) × (1 + 0.20) × (1 + 0.25 + 0.25) + 0
= 700 × 1.20 × 1.50
= 1260
```

## 7. 장비 시스템

### 7-1. 장비 슬롯 (8 슬롯)

| 카테고리 | 슬롯 | 기본 부여 능력치 |
|---------|------|-----------------|
| 무기 | 무기 | 공격력 |
| 방어구 | 모자 | 쿨타임 감소 |
| 방어구 | 상의 | HP |
| 방어구 | 하의 | HP |
| 방어구 | 신발 | 이동속도, HP |
| 방어구 | 장갑 | 크리티컬 데미지 |
| 장신구 | 목걸이 | (랜덤 풀: 쿨감/스킬뎀/크리확) |
| 장신구 | 반지 | (랜덤 풀: 쿨감/스킬뎀/크리확) |

- **캐릭터별 장비**: 캐릭터마다 별도의 장비 세트를 가짐
- **착용 제한 없음**: 다른 캐릭터에도 동일 장비 장착 가능 (이동/공유 자유)

### 7-2. 등급 / 룬 소켓

| 등급 | 색상 | 룬 소켓 수 |
|------|------|-----------|
| 노말 | 회색 | 1 |
| 레어 | 파랑 | 1 |
| 에픽 | 보라 | 2 |
| 유니크 | 주황 | 3 |

### 7-3. 장비 능력치 슬롯 (서브 스탯)

각 장비는 등급에 따라 N개의 **능력치 슬롯**을 가지며, 각 슬롯에는 정해진 풀에서 선택된 스탯이 붙는다. 스탯은 **깡 또는 %** 형태일 수 있다.

| 등급 | 메인 스탯 | 서브 스탯 슬롯 |
|------|----------|---------------|
| 노말 | 1 | 1 |
| 레어 | 1 | 2 |
| 에픽 | 1 | 3 |
| 유니크 | 1 | 4 |

서브 스탯 풀의 정확한 구성은 데이터 작업 시점에 결정한다 (장비 카테고리별로 풀이 다름).

### 7-4. 룬 시스템

룬은 캐릭터의 **공통 스탯과 별개**로 캐릭터 고유 메커니즘을 강화한다.

| 룬 종류 (예시) | 효과 |
|---------------|------|
| 표식 룬 | (Rapier) 표식 최대 중첩 +1 |
| 분쇄 룬 | (Warrior) 대지 분쇄 범위 +20% |
| 잔상 룬 | (Assassin) 잔상 지속 시간 +30% |
| 폭발 룬 | (Ranger) 강화 폭발 화살 데미지 +50% |
| 기본 쿨감 룬 | 차지 스킬 쿨다운 -15% (모든 캐릭터) |
| 회피 룬 | 회피 쿨다운 -20% (모든 캐릭터) |

- 룬은 장비 슬롯에 자유롭게 장착/해제 가능
- 캐릭터마다 사용 가능한 룬이 다를 수 있음

## 8. 스테이지 진행 시스템

### 8-1. 방 배치

```
[보스1] → [인터미션] → [보스2] → [인터미션] → [보스3] → [인터미션] → [보스4]
```

- 보스 사이 3회의 인터미션 방
- 인터미션 방은 **휴식(HP 100% 회복) + 스탯 선택**을 모두 제공

### 8-2. 스탯 선택 풀 (7종)

매번 아래 풀에서 **2개를 랜덤 추출**하여 사용자에게 제시한다. 사용자는 둘 중 1개만 선택한다.

| 스탯 | 1회 효과 | 메타와의 관계 |
|------|----------|--------------|
| 체력 증가 | HP +25% | RunStat % |
| 공격력 증가 | ATK +20% | RunStat % |
| 이동 속도 | MS +15% | RunStat % |
| 회피 쿨다운 감소 | DodgeCDR -20% (2s → 1.6s) | RunStat % |
| 차지 시간 단축 | ChargeTime -15% | RunStat % |
| 무적 시간 증가 | Invincibility +30% | RunStat % |
| 크리티컬 확률 | CritChance +15% | RunStat % |

#### 선택 규칙

- **같은 종류는 한 후보 안에 동시에 안 나옴** (서로 다른 2개 보장)
- **누적 가능**: 다음 인터미션에서 동일 스탯이 다시 나올 수 있고, 사용자가 같은 스탯을 또 선택하면 누적 적용
- **소멸 시점**: 스테이지 클리어 또는 로비 복귀 시 모든 RunStat 초기화
- 강도는 시작점이며, 플레이테스트로 조정

### 8-3. 휴식

- 인터미션 방에 입장하면 **HP가 자동으로 100% 회복**
- 회복은 강제, 스탯 선택은 능동

### 8-4. 사망 / 이어하기

| 상황 | 결과 |
|------|------|
| 보스에게 사망 | 그 보스부터 이어하기 가능 (RunStat 유지) |
| 사용자가 능동적으로 로비 복귀 | 진행도 + RunStat 모두 초기화 |
| 스테이지 클리어 | 보상 화면 → 로비 복귀 / 다음 스테이지 진입 선택 |

## 9. 로비 구조

로비는 **5개의 탭**으로 구성된다. 화면 하단에 가로 버튼으로 배치.

| # | 탭 | 역할 |
|---|----|----|
| 1 | 상점 | 캐릭터/장비 가챠, 충전, 종합 상점 |
| 2 | 캐릭터 관리 | 캐릭터 선택, 장비, 레벨/스킬 강화 |
| 3 | 메인 (홈) | 게임 시작점. 스테이지 표시 + 진입 |
| 4 | 미션 | 일일/주간 미션 확인 및 보상 수령 |
| 5 | 설정 | 게임 설정, 계정 관리 |

### 9-1. 메인 화면 (탭 3)

게임의 시작점. 처음 진입 시 기본 표시 탭.

- 상단 가운데: 현재 도전 중인 스테이지 번호 표시
- 하단: 스테이지 진입 버튼 (눌러서 게임 시작)
- 우측 상단 영역: 우편함 아이콘 (운영 보상, 친구 선물 수령)

### 9-2. 캐릭터 관리 화면 (탭 2)

화면 가운데에 **현재 선택된 캐릭터의 SD 형태 모델**이 표시된다 (인게임과 동일 외형).

캐릭터 관리 화면은 두 개의 서브 영역으로 나뉜다:

#### 9-2-1. 장비 영역

- 캐릭터 좌/우로 8개 장비 슬롯 표시 (착용 중인 아이템)
- 캐릭터 아래쪽 (화면 절반): **인벤토리** 패널
- 인벤토리 탭: 무기 / 방어구 / 장신구 등으로 분류
- 장비 클릭 → 장착, 슬롯 클릭 → 해제 / 룬 관리

#### 9-2-2. 레벨업 / 스킬 영역

- **골드(재화)** 를 사용해 캐릭터 레벨업 → MetaStat 증가
- 캐릭터 스킬 업그레이드도 골드/재료로 진행

### 9-3. 상점 화면 (탭 1)

- 캐릭터 가챠 배너
- 장비 가챠 배너
- 재화 충전 (실결제)
- 상시 상품 / 한정 상품
- 자세한 BM은 §12 참조

### 9-4. 미션 화면 (탭 4)

§10 참조.

### 9-5. 설정 화면 (탭 5)

- 게임 설정 (사운드, 진동, 그래픽)
- 계정 관리 (로그인, 연동, 데이터 백업)
- 약관 / 개인정보 / 고객 센터

## 12. 장비 기믹 및 BM (성장 시스템)

단순 수치 싸움을 배제하고, 적의 패턴을 파훼할 **기믹(유틸리티) 수집**을 과금과 성장의 핵심 동력으로 삼는다.

### 12-1. 속성 기믹 시스템 (총 8종)

보스의 상성(딜량 증가)이 아닌, **액션의 쾌적함**을 담당한다.

| 속성 | 효과 |
|------|------|
| 연타/폭발 (맹독/출혈) | 일정 타수 도달 시 최대 체력 비례 폭발 및 경직(캔슬) |
| 빙결 | 타격 시 보스 패턴 속도 둔화 및 확정 스턴 |
| 흡혈 | 차지/저스트 회피 스킬 적중 시 체력(보호막) 회복 |
| 풍압/바람 | 평타 사거리 증가 및 투사체 상쇄 |
| 전격, 중력, 빛 | (추후 추가 예정) |

### 12-2. 획득 및 로테이션

- **독립 배너**: 캐릭터별 상시 독립 배너 운영. 무기 풀 꼬임 방지.
- **재화 수급**: 무소과금도 인게임 이벤트·파밍으로 티켓 수급. 천장을 통한 명함 획득 및 1~2초월 달성 가능.

### 12-3. 초월 (중복 돌파) — 하이브리드형

| 단계 | 조건 | 효과 |
|------|------|------|
| 명함 (기본) | 무소과금 / 천장 | 코어 기믹 100% 발휘. 액션 쾌감 완벽 보장 |
| 1~2 초월 | 소과금 / 노력 | 기믹 발동 조건 완화 (예: 폭발 필요 타수 감소). 쾌적함 상승 |
| 풀 초월 | 핵과금 | 딜 계수 폭발 및 방어력 무시. 타임어택·랭킹용 스탯 극대화 |

| Phase 10 | Android 빌드 버그 수정 | 런타임 Sprite 직접 생성 방식 교체. 빌드 정상 동작 |
| Phase 11 | 게임 루프 구현 | 결과 UI → 로비 씬 → 게임 씬 전환 정상 동작 |
| **Phase 12-A** | **코어 루프 입력 차단 버그 수정** | **회피/저스트회피/고유스킬/차지스킬 중 Tap 차단 확인** |
| **Phase 12-B** | **로비 5탭 + 캐릭터 관리 + 장비 + 룬 + 미션 + 저장 (메타 시스템)** | **로비 5탭 진입/이탈, 장비 장착·해제, JSON 저장/로드 동작** |
| **Phase 12-C** | **신규 보스 (Specter 완성 + Pyromancer/Berserker/Stormcaller/Gravekeeper/Twin Phantoms)** | **각 보스 단일 보스전에서 정상 플레이 가능** |
| **Phase 12-D** | **스테이지 / 인터미션 방 / 런 스탯 시스템** | **나선비경식 진행, 휴식+스탯 선택, RunStat 누적/소멸 확인** |
| Phase 12-E | 통합 (12-A~D 머지 후 BossRushManager에 신규 보스 등록 + 스테이지 흐름 연결) | 스테이지 1 풀 플레이 가능 |

Phase 12-A ~ 12-D는 **병렬 작업**으로 진행한다. 12-E는 통합 단계로 팀장이 직접 수행한다.
