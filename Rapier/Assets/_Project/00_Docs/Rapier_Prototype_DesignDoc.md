# RAPIER 프로토타입 기획서
**Prototype Design Document v2.0.0**
2026-03-21

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 장르 | 싱글 플레이 실시간 액션 RPG |
| 플랫폼 | 모바일 (Android / iOS), PC 테스트 지원 |
| 타겟 유저 | 지하철·버스 등 통학·출퇴근 시간에 즐기는 숏폼 액션 유저 |
| 화면 방향 | 세로 모드 (Portrait) |
| 렌더 파이프라인 | URP 2D |
| 조작 | 전체 화면, 단일 손가락(엄지) / PC 마우스 지원 |
| 클리어 조건 | 보스 처치 |
| 프로토타입 목표 | 핵심 전투 메커니즘 검증 및 재미 확인 |

### 디자인 철학

- **액션 쾌감**: 단일 손가락 조작으로 완성하는 직관적 전투.
- **명확한 턴**: 이동-공격 간 간섭을 제거한 피지컬 전투.
- **공정한 BM**: 무소과금도 실력으로 극복 가능하되, 과금 시 압도적 쾌적함을 제공하는 구조.

## 2. 게임 구조

### 2-1. 보스 러시 구조

일반 몹 구간을 제거하고 **[보스전 → 정비(보상/상점) → 다음 보스전]** 사이클로만 구성한다.

| 항목 | 내용 |
|------|------|
| 1층 클리어 소요 시간 | 1~3분 |
| 클리어 조건 | 보스 처치 |
| 소탕(Sweep) 시스템 | 타임어택(예: 1분 30초 이내) 클리어 시 소탕 권한 영구 개방. 이후 해당 층 파밍을 10초 컷으로 압축 |

### 2-2. 보스 패턴

모든 보스의 공격은 **시퀀셜 구조**로 정의된다. HP 50% 이하 시 2페이즈 시퀀스로 교체된다.

#### 타이탄 보스

| 페이즈 | 시퀀스 | 설명 |
|--------|--------|------|
| 1페이즈 | Melee → Melee → Melee → 반복 | 전방 부채꼴 근접 공격 |
| 2페이즈 | Melee → Melee → Melee → Charge → 반복 | 3회 근접 후 직선 돌진 |

#### 스펙터 보스

| 페이즈 | 시퀀스 | 설명 |
|--------|--------|------|
| 1페이즈 | Melee → 반복 | 빠른 근접 공격 |
| 2페이즈 | Teleport → Melee → 반복 | 순간이동 후 근접 공격 |

### 2-3. 공격 예고 인디케이터

Windup 구간 동안 아웃라인(범위 경계 고정) + 스캔라인(중심→경계 확장)이 표시된다.
스캔라인이 경계에 닿는 순간 = 공격 발동.

| 모양 | 용도 |
|------|------|
| 부채꼴 (Sector) | 근접 광역 공격 |
| 사각형 (Rectangle) | 돌진 등 직선 범위 공격 |

- `angleOffset` 필드로 여러 방향 인디케이터 동시 표시 가능.
- `lockIndicatorDirection`: true 시 Windup 시작 방향 고정.
- 인디케이터 범위 = 히트 판정 범위 (동일 데이터 사용).

## 3. 조작 체계

입력 유효 영역: **전체 화면**. 단일 손가락(엄지) 기준. PC에서는 마우스로 에뮬레이션.

### 3-1. 기본 입력

| 입력 | 상태 | 조건 | 설명 |
|------|------|------|------|
| Drag | Move | 이동 거리 ≥ 20px, 지속 ≥ 0.3초 | 가상 조이스틱과 함께 이동. 공격 판정 없음 |
| Tap | Attack | 이동 거리 < 20px, 지속 < 0.2초 | 전방 사각형 범위 광역 공격. 즉시 히트 판정. 인디케이터 0.4초 표시. 딜레이 중 연타 차단 |
| Swipe | Dodge | 이동 거리 ≥ 20px, 지속 < 0.3초 | 입력 벡터 방향으로 회피 대시. 대시 전 구간 무적. 쿨다운 2초 |
| Hold → Release | Charge → Skill | 정지 상태, 지속 ≥ 0.3초 | 홀드 중 게이지 충전, 손을 떼면 차지 스킬 발동 |

### 3-2. 저스트 회피 시스템

저스트 회피: **전 캐릭터 공통**. 회피 대시 중 적의 공격을 피격 시 발동. 한 회피당 1회 한정.

| 단계 | 설명 |
|------|------|
| 발동 조건 | 회피 대시 중(JustDodgeAvailable == true) 적 공격 피격 시 발동 |
| 슬로우 모션 | 발동 시 일정 시간 게임 속도 감소 (AnimationCurve 기반, 기본 3초) |
| 카메라 줌 | 발동 시 카메라가 플레이어를 향해 줌인 후 복귀 |
| 무적 구간 | 슬로우모션 구간 전체 무적 유지 |
| 고유 스킬 발동 | 슬로우 중 Hold 입력 시 캐릭터 고유 스킬 즉시 발동 |

## 4. 캐릭터 설계

### 4-1. 공통 스탯 (플레이어)

| 항목 | 수치 | 비고 |
|------|------|------|
| 최대 HP | 500 | 일반 적 10회, 보스 일반 공격 약 6회에 사망 |
| 기본 공격력 | 50 (사각형 범위 광역) | 일반 적 5타, 보스 40타 처치 기준 |
| 이동속도 | 5 | 화면 절반을 약 1초에 이동 |
| 무적 구간 | 회피 대시 전 구간 | unscaledTime 기반 |
| 회피 쿨다운 | 2초 | 쿨다운 중 Swipe 입력 차단. HUD 세로 게이지로 표시 |

### 4-2. 캐릭터별 고유 메커니즘

기본 입력(Tap/Drag/Swipe/Hold)의 동작은 3장 조작 체계를 따른다.
아래는 각 캐릭터가 기본 시스템과 다르게 동작하는 부분만 기술한다.

#### ⚔️ 전사 (Warrior) — '인내와 패링'

- **홀드 중**: 방패 방어 (데미지 감소, 버팀)
- **홀드 후 릴리즈 (차지 스킬)**: 대지 분쇄 — 전방 광역 공격
- **홀드 후 스와이프**: 해당 방향으로 방패 밀쳐내기 (데미지 + 넉백). 피격 중이었다면 **패링(저스트 회피) 판정** 발동
- **저스트 회피(패링) 후 고유 스킬**: 즉시 대지 분쇄 (차지 시간 없이)

#### 🗡️ 암살자 (Assassin) — '잔상 스태킹과 난무'

- **회피**: 일반 회피 동작 동일 (특별 효과 없음)
- **저스트 회피 시 고유 스킬**: 회피 전 위치에 잔상 생성 (피해 없음, 어그로 없음). 잔상 활성 중 본체의 모든 공격(Tap, 차지 스킬)에 잔상이 동시에 동참
- **차지 스킬 (홀드 → 릴리즈)**: 360도 원형 광역 베기

#### 🤺 레이피어 (Rapier) — '빌드업과 수확'

- **회피**: 일반 회피 동작 동일 (특별 효과 없음)
- **저스트 회피 후 고유 스킬**: 가장 가까운 적에게 대시 → 사각형 범위 내 전체 적에게 표식 1중첩 부여 + 데미지 → 회피 목적지(DodgeDest)로 복귀. 표식 최대 5중첩
- **차지 스킬 (홀드 → 릴리즈)**: 표식이 있는 모든 적에게 각 보유 표식 중첩 수 × (공격력 × chargeMarkMultiplier) 데미지. 이후 모든 표식 소비
- **무적 구간 추가**: 스킬 대시~복귀 구간 전체 무적. 스킬 및 회피 진행 중 일반 공격 차단

#### 🏹 사냥꾼 (Ranger) — '거리 조절과 화망'

- **Tap**: 기본 공격이 원거리 사격으로 대체
- **회피**: 입력 벡터 방향으로 대시 + 회피 지점에 지뢰 설치
- **저스트 회피 후 고유 스킬**: 즉시 강화 폭발 화살 발사 (차지 시간 없이)
- **차지 스킬 (홀드 → 릴리즈)**: 일직선 관통 화살. 시전 중 적에게 경직 부여

## 5. 적 설계

### 5-1. 일반 적

| 항목 | 수치 | 비고 |
|------|------|------|
| 최대 HP | 250 | 광역 공격 고려 상향. 단일 기준 플레이어 5타 |
| 공격력 | 50 | 플레이어 10회 피격 사망 기준 |
| 이동속도 | 2.5 | 플레이어의 절반 속도 |
| 공격 주기 | 1.5초 | 타이밍 학습 가능한 주기 |
| 공격 예고 | 0.5초 Windup | 아웃라인 + 스캔라인 인디케이터로 시각 예고 |
| 접근 방식 | 분산 접근 | 각자 다른 각도에서 접근. 뭉치기 방지 |

### 5-2. 보스 (공통)

| 항목 | 수치 | 비고 |
|------|------|------|
| 최대 HP | 2000 | 플레이어 40타 처치. 약 2분 보스전 상정 |
| 일반 공격력 | 80 | 플레이어 약 6회 피격 사망 |
| 광역 스킬 데미지 | 120 | 피격 시 HP 약 24% 감소. 회피 필수 |
| 이동속도 | 1.5 | 느리고 무거운 움직임 |

## 6. 장비 기믹 및 BM (성장 시스템)

단순 수치 싸움을 배제하고, 적의 패턴을 파훼할 **기믹(유틸리티) 수집**을 과금과 성장의 핵심 동력으로 삼는다.

### 6-1. 속성 기믹 시스템 (총 8종)

보스의 상성(딜량 증가)이 아닌, **액션의 쾌적함**을 담당한다.

| 속성 | 효과 |
|------|------|
| 연타/폭발 (맹독/출혈) | 일정 타수 도달 시 최대 체력 비례 폭발 및 경직(캔슬) |
| 빙결 | 타격 시 보스 패턴 속도 둔화 및 확정 스턴 |
| 흡혈 | 차지/저스트 회피 스킬 적중 시 체력(보호막) 회복 |
| 풍압/바람 | 평타 사거리 증가 및 투사체 상쇄 |
| 전격, 중력, 빛 | (추후 추가 예정) |

### 6-2. 획득 및 로테이션

- **독립 배너**: 캐릭터별 상시 독립 배너 운영. 무기 풀 꼬임 방지.
- **재화 수급**: 무소과금도 인게임 이벤트·파밍으로 티켓 수급. 천장을 통한 명함 획득 및 1~2초월 달성 가능.

### 6-3. 초월 (중복 돌파) — 하이브리드형

| 단계 | 조건 | 효과 |
|------|------|------|
| 명함 (기본) | 무소과금 / 천장 | 코어 기믹 100% 발휘. 액션 쾌감 완벽 보장 |
| 1~2 초월 | 소과금 / 노력 | 기믹 발동 조건 완화 (예: 폭발 필요 타수 감소). 쾌적함 상승 |
| 풀 초월 | 핵과금 | 딜 계수 폭발 및 방어력 무시. 타임어택·랭킹용 스탯 극대화 |

## 8. 구현 우선순위 (프로토타입 단계)

| Phase | 내용 | 완료 기준 |
|-------|------|-----------|
| Phase 1 | ServiceLocator / InputState / GestureRecognizer | 입력 4종 판별 및 저스트 회피 타이밍 감지 동작 |
| Phase 2 | 캐릭터 기반 클래스 (Interface, Model, PresenterBase, View) | 플레이스홀더 스프라이트로 MVP 구조 동작 |
| Phase 3 | 플레이어 이동 + 카메라 추적 | Drag → 이동, 화면 추적 확인 |
| Phase 4 | 전투 기반 (CombatSystem, 일반 적, 웨이브 매니저) | 피격 판정, HP 감소, 10초 자동 웨이브 확인 |
| Phase 5 | UI 연결 (HP바, 차지 게이지, Debug 패널) | 전체 루프 플레이 가능 |
| Phase 6 | 전투 고도화 + 저스트 회피 연출 | 슬로우, 무적, 쿨다운, 카메라 줌, HUD 피드백 확인 |
| Phase 7 | 이동 시스템 리팩토링 + 레이피어 고유 메커니즘 | 표식 시스템 + 대시 스킬 + 차지 스킬 동작 확인 |
| Phase 8 | 보스 러시 데모 | 타이탄/스펙터 보스전 플레이 가능 |
| Phase 9 | 공격 인디케이터 + 공격 시퀀서 시스템 | 인디케이터 범위 = 판정 범위 일치. 시퀀셜 패턴 동작 확인 |
| Phase 10 | Android 빌드 버그 수정 | 런타임 Sprite 직접 생성 방식 교체. 빌드 정상 동작 |
| Phase 11 | 게임 루프 구현 | 결과 UI → 로비 씬 → 게임 씬 전환 정상 동작 |
| Phase 12 | 미정 (EnemyStatData CustomEditor / 캐릭터 추가 구현 등) | - |
